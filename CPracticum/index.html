C PRACTICUM: Secret Enciphering / Deciphering of Text

You have 75 minutes to complete this work.

Overview

In this practicum you will implement a simple encipher / decipher system to create and reveal "secret" messages. The encoding is a type of Roman cipher, used during their campaigns in Gaul. Given that this is one of the easiest ciphers to break, calls into question the smarts of the Germanic tribes. 

In our case the cipher works on (and only on) the printable ASCII characters - those whose numeric code is greater than that of the SPACE character and less than that of the DEL character.

How it works

Look at the table to the right and you will see that there are exactly 94 such characters, broken into two halves of 47 characters each along the ASCII table.

A character in the first half (codes 33 through 79) is replaced by the corresponding one in the second half (e.g., the character whose code is greater by 47). Similarly, codes in the second half (80 through 126) are replaced by their correspondents from the first half (e.g., whose codes are 47 smaller).

A useful property of this approach is that enciphered text can be deciphered by using the same algorithm again! Note that newlines and spaces are ignored and not enciphered or deciphered but are instead maintained.  

Example:

Entered text:

Hello, world!

Enciphered text:

w6==@[ H@C=5P


Program Specification

Your program will read a line of characters; no line will have more than 20 characters (ignoring the terminating end-of-line character).

Next you will traverse the array, translating characters according to the algorithm above.

Finally you will print out the translated line. You will do this until the entire file is processed (e.g., when you encounter EOF).

* NOTE: We recommend you begin your implementation by reading the comments in the code and working on the translation functions first!

Instructions

Create a folder called CPracticum/ and download practicum.zip. Here's a description of the files:

  • julius.c- a skeleton program with comments on elements you have to write. Please make sure to read through it all before implementation. 
  • main.c- a skeleton program of your main function for calling from the command line. DON'T FORGET THIS ONE! 
  • julius.h- the header file with your function prototypes and MAXLINESIZE constant. Do not edit.  
  • julius_tests.c- some basic unit tests you should use to work incrementally. Do not edit.  
  • plain.txt- a plain text file for input to your program.
  • cipher.txt - a plain text file with the proper encoding of plain.txt
  • Makefile - a build file for you to compile and run your tests with.
We expect the code to be properly indented, to use #define constants where appropriate, and to write character constants in character notation. You need not add any further documentation.

Building and Testing

  • Create the correct directory and "cd" into that directory.
  • Use "wget" to download the zip file.
  • Use "unzip" to unzip that zip file.
  • Compile and link your code using "make".
    • make
  • To compile and run the unit tests:
    • make test
  • To run the program to process "plain.txt" and compare your output with the correct output:
    • make test_full
  • To debug the unit tests using gdb:
    • Start gdb: gdb run_test
    • Run your tests: r
    • To trace back to where the crash occurred: bt
    • To quit gdb: quit
  • To debug the normal running of the program using gdb:
    • Start gdb: gdb julius
    • To process plain.txt: r <plain.txt >output.txt
    • To trace back to where the crash occurred: bt
    • To quit gdb: quit

Submit your work via Git to /pushbox under the directory "CPracticum" in the root of your repository. You may push other files, but we will grade julius.c and main.c.   As always you are encouraged to push in increments but only your last submission will be used. Your final submission must be entered before end of alloted time. Partial credit will be based on partial behavior (e.g. readline or julius).
Note: very little credit will be given to code that does not compile.
